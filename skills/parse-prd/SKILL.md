# 解析并完善 PRD

通过 MCP 工具 `parse_prd` 解析 PRD 文档，抽取实体并在本体中匹配关系，返回增强后的 PRD。

## 触发条件

- 用户提供一段 PRD 文本或 Markdown
- 用户希望根据已有本体知识为 PRD 补充关系分析
- 用户说“帮我完善这份 PRD”或“分析这个需求文档”

## MCP 工具

- 工具名：`parse_prd`
- 参数：
  - `content`（必填）：PRD 的 Markdown 或纯文本内容
  - `db_path`（可选）：数据库路径，通常不传

## 用法示例

```json
{
  "name": "parse_prd",
  "arguments": {
    "content": "# 用户登录 PRD\n\n用户可以通过手机号登录系统。"
  }
}
```

## 返回值

- `enriched_prd`：增强后的 PRD Markdown，关系信息已**融合到正文**中（而非简单追加附录）
- `summary`：解析摘要，包括抽取实体数、匹配实体数、发现关系数、增强模式

## 增强模式说明

本工具采用**融合增强**模式而非简单的附录追加：

1. **依赖注入**：在关键实体旁标注其依赖关系（如`用户登录（依赖短信验证码服务）`）
2. **影响标注**：标注实体修改可能影响的范围
3. **约束提示**：标注实体受哪些约束限制
4. **包含关系**：标注实体所属的模块

## 注意事项

- 该工具只读分析，不会修改本体数据库
- 需要 LLM 配置正确才能调用
- 增强后的 PRD 保留了原文全部结构和内容，只在关键位置添加标注
